## フェーズ2: ストア用素材の準備

- [ ] **2-1** アプリアイコン 1024×1024 を用意する  
  - すでに `assets/images/app_icon.png` があれば、1024×1024 か確認  
  - App Store Connect の「アプリ情報」でアップロードするときに使う

- [ ] **2-2** スクリーンショットを撮る（実機 or シミュレータ）  
  - **必須サイズ（iPhone 6.7インチ）**: 1290 × 2796 ピクセル（少なくとも2枚、最大10枚）  
  - **推奨: 6.5インチも用意**: 1242 × 2688 ピクセル  
  - 撮る画面の例: ホーム、問題演習、成績、設定 など  
  - キャプチャ: シミュレータなら **Cmd + S** でデスクトップに保存

- [ ] **2-3** スクリーンショットをフォルダに整理する  
  - 例: 「6.7インチ用」「6.5インチ用」などで分けておくと申請時が楽

- [ ] **2-4** 短い説明文（サブタイトル）を書く  
  - 30文字以内（日本語）  
  - 例: 「第1種衛生管理者試験の過去問で爆速対策」

- [ ] **2-5** 詳細説明文を書く  
  - 4000文字以内  
  - 書く内容の例: アプリの機能、こんな人向け、使い方の概要、免責（試験合格は保証しません等）

- [ ] **2-6** キーワードを決める（100文字以内、カンマ区切り）  
  - 例: 衛生管理者,試験,過去問,第1種,資格

- [ ] **2-7** カテゴリを決める  
  - 第1: 例「教育」または「仕事効率化」  
  - 第2: 任意

- [ ] **2-8** 年齢レーティング用の質問に答える準備  
  - App Store Connect で「コンテンツの権利」や「年齢」に関する質問が出るので、内容に合う回答を考えておく（広告あり・課金ありは「はい」など）

## フェーズ3: Apple Developer と Xcode

- [ ] **3-1** Apple Developer Program に登録しているか確認する  
  - https://developer.apple.com/programs/  
  - 未登録なら登録（有料・審査あり）してから先に進む

- [ ] **3-2** Mac に Xcode をインストールする  
  - App Store から「Xcode」で検索してインストール（大きいので時間がかかる）

- [ ] **3-3** ターミナルでプロジェクトの iOS フォルダに移動する  
  - `cd /path/to/eisei_kanrisha_app/ios`

- [ ] **3-4** Xcode でワークスペースを開く  
  - `open Runner.xcworkspace`  
  - または Finder で `ios/Runner.xcworkspace` をダブルクリック

- [ ] **3-5** 左のプロジェクトナビゲーターで「Runner」をクリックする

- [ ] **3-6** 中央の「Signing & Capabilities」タブを開く

- [ ] **3-7** 「Team」で自分の Apple Developer のチームを選ぶ  
  - 出てこない場合は Xcode の **Xcode** → **Settings** → **Accounts** で Apple ID を追加

- [ ] **3-8** 「Automatically manage signing」にチェックが入っているか確認する

- [ ] **3-9** 「Bundle Identifier」を確認する  
  - 例: `com.pizzicato.eiseikanrisha`  
  - 他アプリと被らないこと（変えたい場合はここで変更）

- [ ] **3-10** 上部の実行先を「Any iOS Device (arm64)」に変更する  
  - シミュレータや実機ではなく、この選択が必要

## フェーズ4: ビルドとアップロード

- [ ] **4-1** ターミナルでプロジェクトルートに移動する  
  - `cd /path/to/eisei_kanrisha_app`

- [ ] **4-2** クリーン＆取得する  
  - `flutter clean`  
  - `flutter pub get`

- [ ] **4-3** リリース用にビルドする  
  - `flutter build ipa`  
  - 成功すると `build/ios/ipa/` に IPA ができる

- [ ] **4-4** （別のやり方）Xcode からアーカイブする場合  
  - メニュー **Product** → **Archive**  
  - 完了まで待つ

- [ ] **4-5** アーカイブが終わったら「Distribute App」を選ぶ  
  - **App Store Connect** → **Upload** → 次へ進む  
  - オプションはそのままでOK（Strip symbols など）

- [ ] **4-6** アップロードが完了するまで待つ  
  - 成功すると「Upload Successful」などと表示される

- [ ] **4-7** メールで「ビルドが処理されました」という通知が来るまで待つ（数分〜数十分）  
  - これが出たら App Store Connect にビルドが表示される

## フェーズ5: App Store Connect で申請

- [ ] **5-1** https://appstoreconnect.apple.com/ にログインする

- [ ] **5-2** 「マイ App」をクリックする

- [ ] **5-3** アプリがまだなければ「＋」→「新規 App」  
  - プラットフォーム: iOS  
  - 名前: 例「衛生管理者 爆速合格」  
  - 主言語: 日本語  
  - Bundle ID: Xcode で設定したもの（例 com.pizzicato.eiseikanrisha）を選択  
  - SKU: 任意の英数字（例 eisei1）

- [ ] **5-4** 左メニュー「App 情報」を開く  
  - プライバシーポリシー URL: `https://so0727.github.io/eisei-kanrisha-app/`  
  - カテゴリ（第1・第2）を選択  
  - 必要ならサブタイトルなどもここで設定

- [ ] **5-5** 左メニュー「1.0 の準備」を開く（または「iOS App」のバージョン「1.0」）

- [ ] **5-6** 「ビルド」の「＋」をクリックし、アップロードしたビルドを選択する  
  - まだ表示されていなければ、4-7 のメールを待つ

- [ ] **5-7** 「スクリーンショット」に 6.7インチ用（1290×2796）をアップロードする  
  - 最低2枚。ドラッグ＆ドロップで追加

- [ ] **5-8** 「説明」に詳細説明文を貼り付ける

- [ ] **5-9** 「キーワード」にキーワードを入力する（カンマ区切り）

- [ ] **5-10** 「サブタイトル」に短い説明（30文字以内）を入力する

- [ ] **5-11** 「宣伝用テキスト」（任意）があれば入力する

- [ ] **5-12** 「ビルド」が選択された状態になっているか確認する

- [ ] **5-13** 「輸出コンプライアンス」「コンテンツの権利」「広告識別子」などの質問に答える  
  - 広告（AdMob）を使っているなら「広告が含まれています」は「はい」

- [ ] **5-14** 左メニュー「料金と配信」を開く  
  - 無料 or 有料を選択  
  - 配信国・地域を選択（日本など）

- [ ] **5-15** すべて入力できたら「審査に提出」ボタンを押す

- [ ] **5-16** 最後の確認画面で「提出」を押す

## フェーズ6: 提出後

- [ ] **6-1** ステータスが「審査待ち」になっているか確認する

- [ ] **6-2** 審査結果のメールを待つ（通常 1〜3 日程度）

- [ ] **6-3** 却下された場合はメールの理由を読み、該当箇所を直して再提出する

- [ ] **6-4** 承認されたら「リリース」を手動で行うか、自動リリースの設定にする  
  - 「審査通過後に自動でリリース」にしている場合はそのままストアに掲載される
